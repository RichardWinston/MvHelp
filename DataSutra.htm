SUTRA (Version 2D3D.1) Data Requirements

Date requirements for visualizing SUTRA (Version 2D3D.1) results are as follows:

  • The finite-element mesh, as defined by the input parameter MSHSTR, must be 3D REGULAR or 2D REGULAR.
  • Nodewise results must be save in a "nod" file. For a 3D mesh, the "nod" file must contain 4, 5 or 6 columns of data in the order specified below. These columns are (See sample "nod" file.):
    1. x coordinate of node (required)
    2. y coordinate of node (required)
    3. z coordinate of node (required)
    4. pressure, concentration, temperature, or saturation at node (required)
    5. pressure, concentration, temperature, or saturation at node (optional)
    6. pressure, concentration, temperature, or saturation at node (optional)
    For a 2D mesh, the z coordinate is not needed.
  • To view velocity vectors, elementwise results must be saved in an "ele" file. For a 3D mesh, the "ele" file must contain 6 columns of data in the order specified below. These columns are (See sample "ele" file):
    1. x coordinate of element center
    2. y coordinate of element center
    3. z coordinate of element center
    4. x component of velocity at element center
    5. y component of velocity at element center
    6. z component of velocity at element center
    • For a 2D mesh, the z coordinate is not needed.
